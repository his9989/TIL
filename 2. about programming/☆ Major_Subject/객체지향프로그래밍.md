### Call by
* call by reference : 함수 호출 시 참조 전달
<code>
      
      void swap(int &a, int &b){
            int tmp;
            tmp = a;
            a = b;
            b = tmp;
            
      }
      int main(){
            int m = 2, n = 9;
            swap(m, n);
      }
      
</code>
* call by value : 변수 전달
<code>
      
      void swap(int a, int b){
            int tmp;
            tmp = a;
            a = b;
            b = tmp;
      }
      int main(){
            int m = 2, n = 9;
            swap(m, n);
      }
      
</code>

* call by address : 포인터 전달
<code>
      
      void swap(int *a, int *b){
            int tmp;
            tmp = *a;
            *a = *b;
            * b = tmp;
      }
      int main(){
            int m = 2, n = 9;
            swap(&m, &n);
      }
      
</code>

Power operator+ (int op1, Power op2){
  Power tmp;
  tmp.kick = op1 + op2.kick;    // kick, punch를 접근하기 위해 연산자함수 operator+는 friend로 구현해야된다.
  tmp.punch = op1 + op2.punch;  // kick, punch를 접근하기 위해 연산자함수 operator+는 friend로 구현해야된다.
  return tmp;
}

8. 업캐스팅, 다운 캐스팅 (190502)
Up 캐스팅 : 파생 클래스 포인터가 기본 클래스 포인터에 치환되는 것

ex) 부모 클래스의 포인터로 자식 클래스 중 기본클래스 멤버만 접근하는 경우
포인터에 대한 포인터라고 할 수 있다.
즉, 기본 클래스 멤버만 가능하다.

ColorPoint cp; // 기본 클래스
ColorPoint *pDer = &cp; // 파생 클래스
Point* pBase = pDer; // 업캐스팅. pBase로는 cp의 public 멤버만 접근할 수 있다. (P13)

Down 캐스팅 : 기본 클래스의 포인터가 파생 클래스의 포인터에 치환되는 것
강제 타입 변환이 필요하다

ColorPoint cp;
ColorPoint *pDer;
Point* pBase = &cp; // 업캐스팅
pDer = (ColorPoint *)pBase; // 다운캐스팅

소멸자는 반대의 순서로 호출된다.
컴파일러는 묵시적으로 기본 클래스의 기본 생성자를 호출하도록 한다.

9. 동적 바인딩
Virtual Function : 동적 바인딩 지시어. 함수에 대한 호출 바인딩 실행 전까지 아무런 동작 X

오버라이딩하면 업캐스팅해줘도 자녀 함수만 호출할 수 있다.
함수 재정의와 오버라이딩 용어 정리

virtual(동적 바인딩) 사용하는 경우 : 오버라이딩
정적 바인딩의 경우 : 함수 재정의
자바는 무조건 동적 바인딩이 일어난다.
P9 오버라이딩 활용하는 방법. 소공에서 쓸만할 듯

즉, 동적 바인딩이란 :

부모 클래스에 대한 포인터로 가상함수를 호출하는 경우,
객체 내에 오버라이딩한 자녀 클래스의 함수를 찾아서 자녀 클래스의 함수를 실행한다.
